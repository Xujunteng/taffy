## 二、完整 API 接口文档

### 统一规范

- **Base URL**: `http://localhost:5173`（开发时经 Vite 代理转发）
- **Content-Type**: `application/json`（文件上传除外）
- **认证方式**: Header 中携带 `Authorization: Bearer <token>`
- **响应格式**:
```json
{
  "code": 200,       // 200=成功, 400=参数错误, 401=未登录, 404=不存在, 500=服务器错误
  "message": "success",
  "data": { }        // 具体数据
}
```

### 模块 A：用户认证 (主后端 :8081)

负责服务：**backend-main**

| # | 方法 | 路径 | 认证 | 请求参数 | 响应 data | 说明 |
|---|------|------|------|----------|-----------|------|
| A1 | `POST` | `/api/auth/register` | 否 | `{ username, password, email }` | `null` | 用户注册，密码BCrypt加密 |
| A2 | `POST` | `/api/auth/login` | 否 | `{ username, password }` | `{ token, userId, username }` | 登录返回JWT Token |
| A3 | `GET` | `/api/auth/userinfo` | 是 | — | `{ id, username, email, role }` | 获取当前用户信息 |

**实现文件**: `backend-main/src/main/java/com/taffy/controller/AuthController.java`

### 模块 C：音频上传与声音训练 (语音AI :8082)

## 四、数据库表结构

```sql
-- schema.sql 位于 backend-main/src/main/resources/schema.sql

-- 用户表（成员2维护）
CREATE TABLE IF NOT EXISTS users (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    email VARCHAR(100),
    role VARCHAR(20) DEFAULT 'user',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 声音模型表（成员2维护，成员3读写）
CREATE TABLE IF NOT EXISTS voice_models (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    name VARCHAR(100) NOT NULL,
    description TEXT,
    status VARCHAR(20) DEFAULT '训练中',  -- 训练中/就绪/失败
    audio_file_path VARCHAR(500),
    model_params TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- TTS任务表（成员3维护）
CREATE TABLE IF NOT EXISTS tts_tasks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    voice_model_id INTEGER,
    text_content TEXT,
    audio_output_path VARCHAR(500),
    status VARCHAR(20) DEFAULT 'pending',  -- pending/processing/completed/failed
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (voice_model_id) REFERENCES voice_models(id)
);

-- 脚本表（成员4维护）
CREATE TABLE IF NOT EXISTS scripts (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    category VARCHAR(50),
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id)
);

-- 评价表（成员4维护）
CREATE TABLE IF NOT EXISTS feedbacks (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    voice_model_id INTEGER,
    rating INTEGER CHECK(rating >= 1 AND rating <= 5),
    comment TEXT,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (voice_model_id) REFERENCES voice_models(id)
);

-- 直播会话表（成员4维护）
CREATE TABLE IF NOT EXISTS live_sessions (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    script_id INTEGER,
    platform VARCHAR(50),
    start_time DATETIME,
    end_time DATETIME,
    stats_data TEXT,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (script_id) REFERENCES scripts(id)
);

-- 帮助文章表（成员4维护）
CREATE TABLE IF NOT EXISTS help_articles (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title VARCHAR(200) NOT NULL,
    content TEXT,
    category VARCHAR(50),
    sort_order INTEGER DEFAULT 0,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 六、跨成员协作要点

### 服务间调用关系

```
前端 (Vue 3 :5173)
  │
  ├── /api/auth/*     → backend-main (:8081)
  ├── /api/voices/*   → backend-main (:8081)
  ├── /api/audio/*    → backend-voice-ai (:8082)
  ├── /api/tts/*      → backend-voice-ai (:8082)
  ├── /api/voice/*    → backend-voice-ai (:8082)
  ├── /api/feedback/* → backend-extended (:8083)
  ├── /api/stats/*    → backend-extended (:8083)
  ├── /api/scripts/*  → backend-extended (:8083)
  └── /api/help/*     → backend-extended (:8083)
```

### 数据流

```
用户注册/登录（成员2: UserService）
  → 上传音频（成员3: AudioController）
    → 创建声音模型记录（成员2的 voice_models 表，成员3直接写）
    → 调用阿里云训练（成员3: TrainController）
      → 更新声音模型状态为"就绪"（成员3）
        → TTS合成（成员3: TTSController）
          → 存储合成结果到 tts_tasks 表（成员3）
          → 返回音频给前端播放（成员1）

评价系统（成员4: FeedbackController）
  → 读取 voice_models 表（由成员2/3写入）
脚本管理（成员4: ScriptsController）
  → 脚本可用于直播面板TTS（成员1前端选择脚本内容发TTS请求）
```

### 协作关键点

1. **共享数据库**: 三服务共享 `./data/taffy.db`，需要互相了解对方写入的表
2. **认证统一**: 三个服务的认证拦截器逻辑相同，JWT密钥需一致（`application.yml` 中 `jwt.secret` 各服务独立配置）
   - 当前前端框架中，语音AI和拓展功能服务**没有认证拦截器**（只有主后端有）
   - **成员2需要把 `JwtUtil.java` 和 `AuthInterceptor.java` 复制给成员3和成员4**，或者成员3/4自行实现
3. **API 路径不冲突**: 已通过不同前缀分配给不同服务
4. **前端对接**: 成员1的API调用路径已写好（`src/api/*.js`），后端实现时需保证路径完全一致
5. **Git 协作**: 
   - 建议每人使用独立分支开发自己的模块
