### D2 已完成：API 完善与安全加固

#### 2.1 用户认证接口 (AuthController) ✅

| 端点 | 状态 | 说明 |
|------|------|------|
| `POST /api/auth/register` | ✅ 完善 | 已添加完整参数校验 |
| `POST /api/auth/login` | ✅ 完善 | 已添加非空校验 |
| `GET /api/auth/userinfo` | ✅ 完善 | 已排除 password_hash 返回 |

**注册校验规则**:
- 用户名：3-20 字符，只允许 `[a-zA-Z0-9_]`
- 密码：至少 6 位
- 邮箱：可选，提供时校验格式 `xxx@xxx.xx`
- 用户名唯一性由 Service 层保证

**安全增强**:
- 密码使用 BCrypt 加密存储
- `getUserInfo` 返回前自动清除 `passwordHash` 字段（UserService 处理）
- 用户不存在时返回 404 而非空数据

#### 2.2 声音管理接口 (VoiceController) ✅

| 端点 | 状态 | 说明 |
|------|------|------|
| `GET /api/voices` | ✅ 完善 | 仅返回当前用户的声音模型 |
| `GET /api/voices/{id}` | ✅ 完善 | 已添加所有权验证 (403) |
| `PUT /api/voices/{id}` | ✅ 完善 | 已添加所有权验证 (403) |
| `DELETE /api/voices/{id}` | ✅ 完善 | 已添加所有权验证 + 音频文件物理删除 |

**所有权验证**:
- 所有非列表接口均验证 `voice.userId == request.userId`
- 不匹配返回 `403 - 无权访问/修改/删除该声音模型`

**删除增强**:
- 删除声音模型时同步删除关联音频文件（`VoiceModelService.deleteVoice`）
- 通过 `java.io.File.delete()` 物理删除
- 删除失败时记录 warn 日志但不阻断数据库删除

#### 2.3 全局异常处理 (GlobalExceptionHandler) ✅

| 异常类型 | HTTP状态 | code | 处理方式 |
|----------|----------|------|----------|
| `RuntimeException` | 200 | 400 | 返回业务错误消息 |
| `IllegalArgumentException` | 200 | 400 | 返回参数错误消息 |
| `Exception` | 200 | 500 | 返回"服务器内部错误" |

基于 `@RestControllerAdvice` 实现，统一异常响应格式为 `{ code, message, data }`。

### D3 已完成：JWT 与拦截器 ✅

#### 3.1 JWT 配置 (`config/JwtUtil.java`)
- 密钥: `taffy-live-assistant-jwt-secret-key-2026`（由 `application.yml` 配置）
- 过期时间: 86400000ms = 24小时
- 算法: HMAC-SHA (jjwt 0.12.x)
- Token 载荷: `sub=userId`, `username`, `iat`, `exp`

#### 3.2 认证拦截器 (`config/AuthInterceptor.java`)
- 放行: `OPTIONS` 预检请求、`/api/auth/login`、`/api/auth/register`
- 拦截: 其他所有 `/api/**` 路径
- 前置处理: 从 Header 提取 `Authorization: Bearer <token>`，解析 userId 存入 `request.setAttribute("userId", userId)`

#### 3.3 CORS 配置 (`config/WebConfig.java`)
- 允许所有来源 `allowedOriginPatterns("*")`
- 允许方法: GET, POST, PUT, DELETE, OPTIONS
- 允许所有 Headers
- 允许携带 Cookie (`allowCredentials(true)`)

## 七、开发注意事项

1. **启动顺序**: 主后端必须先启动（负责建表），语音AI和拓展功能后启动
2. **JWT 密钥**: 三个服务的 `jwt.secret` 必须完全一致
3. **外键约束**: SQLite 默认不启用外键，通过 `PRAGMA foreign_keys = ON` 在每个连接上手动启用
4. **schema.sql 幂等**: 所有 DDL 使用 `CREATE TABLE IF NOT EXISTS`，支持重复执行
5. **文件编码**: 所有源文件使用 UTF-8
6. **端口占用**: 确保 8081 端口未被其他进程占用
